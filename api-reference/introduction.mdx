## URL base

```
https://api.scrivot.cl
```

---

## Autenticación

Todos los endpoints privados requieren un JWT en el encabezado `Authorization`:

```http
Authorization: Bearer <token_de_servicio>
```

El token de servicio se obtiene en `POST /auth/login` o `POST /auth/signup`.

<Note>
  Los endpoints marcados como **Público** no requieren autenticación.
</Note>

---

## Límite de solicitudes

La API tiene los siguientes límites por dirección IP:

| Endpoint | Límite |
|---|---|
| Todos los endpoints | 120 solicitudes/min |
| `POST /auth/login` | 10 solicitudes/min |
| `POST /auth/signup` | 5 solicitudes/min |
| `POST /auth/forgot-password` | 5 solicitudes/min |
| `POST /auth/resend-confirmation` | 5 solicitudes/min |

Al superar el límite, la API retorna `429 Demasiadas solicitudes` con el encabezado `Retry-After: 60`.

---

## Formato de respuesta

Las respuestas son JSON. Los campos de fecha usan formato **ISO 8601**:

```json
{
  "id": 1,
  "name": "Mi chatbot",
  "created_at": "2025-01-15T10:30:00.000Z",
  "updated_at": "2025-04-29T14:22:00.000Z"
}
```

---

## Errores

Los errores siguen un formato consistente:

```json
{
  "statusCode": 403,
  "message": "No tienes permisos para acceder a este recurso",
  "error": "Forbidden",
  "code": "INSUFFICIENT_PERMISSIONS"
}
```

| HTTP | Significado |
|---|---|
| 400 | Solicitud inválida — parámetros incorrectos |
| 401 | No autorizado — token inválido o expirado |
| 403 | Prohibido — sin permisos suficientes |
| 404 | No encontrado — el recurso no existe |
| 429 | Demasiadas solicitudes — límite de velocidad alcanzado |
| 500 | Error interno del servidor |

---

## Aislamiento por espacio de trabajo

Todos los datos están **aislados por espacio de trabajo (workspace)**. Un usuario solo puede acceder a los recursos del suyo propio. Los superadmins (`role_id: 1`) tienen acceso a todos los workspaces.
