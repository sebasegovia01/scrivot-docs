## Tipos de tokens

Scrivot usa tres tipos de credenciales:

### 1. Token de servicio (JWT de usuario)

Autenticación estándar para el dashboard y la API.

| Campo | Valor |
|---|---|
| **Tipo** | JWT Bearer |
| **Expiración** | 24 horas |
| **Uso** | `Authorization: Bearer <token>` |
| **Obtención** | `POST /auth/login` o `POST /auth/signup` |

**Contenido del token:**
```json
{
  "sub": 42,
  "userId": 42,
  "email": "usuario@empresa.cl",
  "workspace_id": 7,
  "role_id": 2,
  "type": "user"
}
```

---

### 2. Token de widget (JWT del widget)

Token especial para el widget embebido. No expira y está vinculado a un chatbot específico.

| Campo | Valor |
|---|---|
| **Tipo** | JWT Bearer |
| **Expiración** | Nunca (hasta que se revoque) |
| **Uso** | Parámetro `?jwt=<token>` en el script del widget |
| **Obtención** | Automático al crear un chatbot, o `POST /auth/generate-widget-token` |

**Contenido del token:**
```json
{
  "userId": 42,
  "email": "usuario@empresa.cl",
  "workspace_id": 7,
  "chatbot_id": 3,
  "role_id": 2,
  "type": "widget"
}
```

<Warning>
  El token de widget es **público** (va embebido en tu HTML). No incluye información sensible, pero está vinculado a tu workspace y chatbot. No lo reutilices entre chatbots distintos.
</Warning>

---

### 3. Clave de API

Reservada para integraciones futuras. Se genera al registrarse pero actualmente no está activa en los endpoints.

## Roles

| `role_id` | Nombre | Descripción |
|---|---|---|
| `1` | Superadmin | Acceso total a todos los workspaces |
| `2` | Admin | Acceso completo a su workspace |
| `3` | Agente | Acceso limitado (sin configuración) |

---

## Seguridad

- Todos los endpoints privados requieren el encabezado `Authorization: Bearer <token>`
- Los endpoints públicos no requieren autenticación
- Límite de solicitudes por IP en los endpoints de acceso:
  - Inicio de sesión: **10 intentos/minuto**
  - Registro: **5 intentos/minuto**
  - Recuperar contraseña: **5 intentos/minuto**
